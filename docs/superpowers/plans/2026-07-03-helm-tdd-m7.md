# helm-tdd M7 Implementation Plan — Observable, fail-closed runs + eval-gated pattern embedding

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking. In this brain, per-task implementation is dispatched to **ask-cursor (Composer 2.5)** under the gate below, per Rule #0; the orchestrator reviews each task's diff.

**Goal:** Make helm-tdd runs observable (per-phase cost/denials/raw output in ledger+artifacts), make the mutation gate fail-closed, give js/go the constant mutant, bound REFACTOR to the slice scope, and build the eval harness that decides whether practicing-tdd pattern guidance earns a place in the RED prompt.

**Architecture:** All changes ride existing seams: telemetry extends `RunPhaseResult` and threads through the FSM into the ledger/artifact vault; mutation outcomes get a three-way discriminant consumed by the FSM; the constant mutant reuses each runner's existing AST machinery (static extraction, no target-code execution); REFACTOR bounding is a post-hoc diff check against AST function spans with revert-on-violation. The eval harness drives `runFeature` directly (validated direct-driver pattern) with an env-var prompt seam for arm B.

**Tech Stack:** Same as M0-M6 — TypeScript 5.6 strict, Node 20, Vitest 4; embedded python `ast` / go `go/ast` scripts run with the target toolchain; TypeScript compiler API for js. **No new dependencies.**

**Spec:** `docs/superpowers/specs/2026-07-03-helm-tdd-m7-design.md` (approved). Build order and policies (fail-closed constant gate, revert+record bounding, helpers allowance, non-compiling-mutant discard) are decided there — do not re-litigate.

```ts
// gates/refactorGate.ts
export interface RefactorScopeViolation { offendingHunks: string[] }  // "@@ -a,b +c,d @@" headers
// RefactorAttemptResult gains: scopeViolation: RefactorScopeViolation | null;
//                              scopeCheck: "enforced" | "skipped_no_spans";
// SliceExecutionResult gains:  refactorScopeViolation: RefactorScopeViolation | null;
```

- [ ] **Step 1 (allowed region):** in `attemptRefactor`, BEFORE the backend call, capture `preSpans = await runner.functionSpans(implPath)`. AFTER the backend call + diff-guard + test-pass check (insert after the existing test-pass revert, before the python ratchet), compute `postSpans`. If either span set is empty → `scopeCheck: "skipped_no_spans"`, no bounding (fail-soft; diff-guard and tests remain the hard gates). Else:
  - `sliceSpan` = postSpans entry named `functionName` (missing → skipped_no_spans).
  - `newFns` = postSpans whose names are not in preSpans.
  - Allowed helper spans: iterate to fixpoint — a new fn is allowed if its name appears as a word (`\b<name>\b` regex, escape the name) in the source text of the slice span or of an already-allowed helper span.
  - `allowed` = sliceSpan ∪ allowed helper spans (line ranges).
- [ ] **Step 2 (hunk check):** `git diff -U0 HEAD -- <implRelPath>` in `targetDir` (GREEN is committed before REFACTOR runs, so HEAD diff == the refactor's changes exactly). Parse `@@ -a,b +c,d @@` headers. A hunk is in-bounds iff its new-side range `[c, c+max(d,1)-1]` lies entirely within `allowed` (pure deletions, `d === 0`, use line `c` — the line before the deletion — with +1 tolerance). Any out-of-bounds hunk → restore `originalSource` (the pre-refactor snapshot already held), return `applied: false`, `scopeViolation: { offendingHunks }`, reason `"refactor exceeded slice scope; reverted"`.
- [ ] **Step 3 (prompt):** `buildRefactorPrompt` in `featureFsm.ts` gains: `"Confine your changes to ${slice.functionName} and any new helper functions you extract from it -- edits elsewhere in the file will be reverted."`
- [ ] **Step 4 (ledger):** thread `scopeViolation` into `SliceExecutionResult.refactorScopeViolation` (all push sites; null where refactor didn't run); on violation also `writeArtifact(artifactRoot, runId, `slice-${i}-refactor-scope`, { offendingHunks, allowed })`.
- [ ] **Step 5 (tests):** with a ScriptedBackend refactor step that (a) edits only inside the function → applied, no violation; (b) also renames a variable in ANOTHER function → reverted, `scopeViolation.offendingHunks.length > 0`, file content back to pre-refactor; (c) extracts a helper called from the slice function → applied (helpers allowance); (d) runner with `functionSpans → []` → `scopeCheck: "skipped_no_spans"`, refactor applied as today.
